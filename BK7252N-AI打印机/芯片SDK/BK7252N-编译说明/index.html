    BK7252N 固件编译说明¶

    本文档说明本仓库（bk7252n_sdk-SDK_3.0.76.1_witch_lcd，RT-Thread + Beken BK7252N）在 Linux 下的编译命令与从配置到产物的完整流程。烧录步骤见同仓库 [[项目/BK7252N-AI打印机/开发记录/指南/bk7252n-烧录教程]]。

    1. 环境准备¶

    1.1 必备软件¶

    项目 说明
    SCons 工程使用 SCons 作为构建系统
    Python SCons 与 tools/scripts/post_action.py 依赖 Python
    ARM GCC 工具链 arm-none-eabi-gcc 等，版本需与 rtconfig.py 中 SDK_TOOLCHAIN（当前为 5.4.1）一致

    1.2 工具链路径¶

    根目录 rtconfig.py 默认将 EXEC_PATH 设为：

    /opt/gcc-arm-none-eabi-5_4-2016q3/bin

    若工具链安装在其他目录，请设置环境变量 RTT_EXEC_PATH 指向该 bin 目录（末尾带不带 / 一般均可，以本机实测为准）。

    可选：

    • RTT_ROOT：若 RT-Thread 不在工程默认相对路径 rt-thread，可通过此变量指定。

    1.3 工作目录¶

    所有编译命令均在工程根目录（包含 SConstruct、rtconfig.py 的目录）下执行：

    cd /path/to/bk7252n_sdk-SDK_3.0.76.1_witch_lcd

    2. 核心编译命令（BK7252N）¶

    日常全量编译固件（并行数按 CPU 调整）：

    scons --beken=bk7252n -j8

    等价变体示例：

    scons --beken=bk7252n -j4

    说明：

    • --beken=bk7252n 必选，用于选择 BK7252N 平台并触发对应的配置与链接脚本（见下文流程）。
    • -jN 为 SCons 并行任务数，可按机器核数修改。

    3. 编译过程（从头到尾）¶

    整体可概括为：按芯片切换 BSP 配置 → SCons 编译链接生成 ELF/_bin → 打包工具生成烧录镜像并整理到 out/。

    3.1 解析 --beken 并生成平台配置¶

    SCons 读取 rt-thread/tools/building.py 时，在常规固件编译路径下会执行：

    1. 调用 generate_sys_config（Linux 为 tools/scripts/generate_sys_config.sh）
      对 bk7252n 会：
    2. 将 beken378/app/config/sys_config_bk7252n.h 同步为根目录 config/sys_config.h（内容变化时更新）；
    3. 将 config/rtconfig/rtconfig_bk7252n.h 同步为根目录 rtconfig.h；
    4. 用预处理器展开 config/bk7252n_boot.lds 生成工程根目录下的 link.lds（实际链接脚本）；

    5. 写入 .platform 文件，内容为 bk7252n，标记当前目标芯片。

    6. 读取 Flash 容量
      从 config/sys_config.h 中解析 CFG_FLASH_SELECTION_TYPE 及其对应数值（例如 4M 对应 0x400000），用于后续打包选择分区 JSON 模板。

    7. 追加 POST_ACTION
      在链接完成、objcopy 生成 rtthread.bin 之后，追加执行：

    python tools/scripts/post_action.py bk7252n <flash_size>

    3.2 SCons 编译与链接¶

    • 根 SConstruct 使用 RT-Thread 的 PrepareBuilding / DoBuilding 收集各子目录 SConscript，编译得到目标 rtthread.elf。
    • rtconfig.py 中 POST_ACTION（在 Beken 逻辑追加 pack 之前）包含：将 rtthread.elf 转为 rtthread.bin，并执行 size 等。

    BK7252N 闭源/预编译库（如 libip_7252n_*、libble_7252n_* 等）由 Beken 侧 SConscript 按芯片型号选择，编译日志中会出现 using ... libxxx_7252n.a 一类提示。

    3.3 打包与输出目录（post_action.py）¶

    tools/scripts/post_action.py 根据 芯片名 与 Flash 大小：

    1. 生成 config.json：bootloader 使用 tools/beken_packager/bootloader_bk7252n_uart1_v1.0.14.bin，app 段指向 rtthread.bin；
    2. 调用 tools/beken_packager/beken_packager（Linux 下为 ./tools/beken_packager/beken_packager）生成完整升级包与 UART 包等；
    3. 清空并重建 out/，将 rtthread.bin、rtthread.elf、rtthread.map、合成镜像、bootloader 副本等移入其中。

    当前 sys_config_bk7252n.h 中典型配置为 4M Flash（CFG_FLASH_SELECTION_TYPE → FLASH_SELECTION_TYPE_4M），产物命名中会带有 4M 版本字段（具体文件名以构建输出为准，常见形式包括 all_4M.*.bin、rtthread_uart_4M.*.bin 等）。

    4. 常用辅助命令¶

    用途 命令
    清理构建 scons -c 或 scons --clean（习惯上与 --beken=bk7252n 同用）
    图形/菜单配置（RT-Thread BSP） scons --beken=bk7252n --menuconfig
    仅重建/清理某类 Beken 静态库（较少用） scons --beken=bk7252n --buildlib=<name> / --cleanlib=<name>
    批量构建该平台全部库 scons --beken=bk7252n --buildlibs
    查看 SCons 与本工程选项 scons --beken=bk7252n --help

    注意： --buildlibs / --cleanlibs 会在脚本内多次调用 scons --buildlib=...，首次完整 SDK 搭建或切换大版本时可参考；日常改应用代码一般只需 scons --beken=bk7252n -jN。

    5. 与配置、产物相关的路径速查¶

    内容 路径
    BK7252N 芯片系统配置（源） beken378/app/config/sys_config_bk7252n.h
    编译时使用的副本 config/sys_config.h（由脚本从上面复制）
    RT-Thread 配置头（源） config/rtconfig/rtconfig_bk7252n.h → 根目录 rtconfig.h
    链接脚本源 config/bk7252n_boot.lds → 预处理后 link.lds
    最终固件与 map 等 out/
    打包工具与 Bootloader 二进制 tools/beken_packager/
