    2026-05-18 模式图标 BMP24→RGB565 压缩优化开发记录¶

    需求背景¶

    首页六模式图标（动漫、卡通、水墨画、线条画、简笔画、速写）原始采用 BMP24 格式嵌入 C 数组（LV_IMG_CF_RAW），每张 100×100 图片约 30 KB，六张合计约 180 KB 像素数据（C 源码文件约 1.1 MB）。在 BK7252N 这种资源受限的 MCU 上，Flash 和内存占用较大，需要进行压缩优化。

    优化目标¶

    • 将六张模式图标的颜色格式从 BMP24（3 字节/像素） 转换为 RGB565（2 字节/像素）
    • 去除 BMP 文件头（54 字节/张），输出为裸 RGB565 像素数组
    • LVGL 颜色格式从 LV_IMG_CF_RAW 切换为 LV_IMG_CF_TRUE_COLOR
    • 不影响显示效果（100×100 尺寸下 RGB565 视觉无差异）

    涉及文件¶

    文件 说明
    applications/images/mode_icons_bmp.c 优化后：RGB565 裸像素数组，约 724 KB
    applications/images/mode_icons_bmp.c.bak 优化前备份：BMP24 完整格式，约 1.08 MB
    applications/images/mode_icons_bmp.h 头文件（无需修改，接口不变）
    applications/images/convert_bmp_to_rgb565.py 新增：一键转换脚本

    技术方案¶

    源格式（BMP24）¶

    BMP Header (54 bytes) + BGR888 pixels (bottom-up, row-padded to 4-byte)
每像素 3 字节：B, G, R
100×100 → 行字节 = ((100 * 3 + 3) / 4) * 4 = 304 bytes
总数据 ≈ 54 + 304 * 100 = 30,454 bytes/张

    目标格式（RGB565）¶

    裸像素数组 (top-down, no padding)
每像素 2 字节：RGB565 (R5+G6+B5)
100×100 → 100 * 100 * 2 = 20,000 bytes/张
节省 ≈ 10,454 bytes/张 (34.3%)

    转换算法¶

    def bgr888_to_rgb565(b, g, r):
    """BGR888 (BMP byte order) → RGB565"""
    return ((r >> 3) << 11) | ((g >> 2) << 5) | (b >> 3)

    关键步骤： 1. 解析 BMP 头，获取宽高、色深、像素偏移 2. 校验为 24-bit BMP 3. 从底行到顶行（BMP 存储顺序），逐像素做 BGR→RGB565 转换 4. 输出为 top-down 裸字节数组 5. 生成 LVGL 兼容的 C 数组代码，header.cf = LV_IMG_CF_TRUE_COLOR

    LVGL 描述符变更¶

    // 优化前
const lv_img_dsc_t mode_icon_anime_bmp = {
    .header.cf = LV_IMG_CF_RAW,         // BMP24 原始格式
    .header.w = 100,
    .header.h = 100,
    .data_size = 30054,
    .data = mode_icon_anime_bmp_map,
};

// 优化后
const lv_img_dsc_t mode_icon_anime_bmp = {
    .header.cf = LV_IMG_CF_TRUE_COLOR,   // RGB565 标准格式
    .header.w = 100,
    .header.h = 100,
    .data_size = 20000,
    .data = mode_icon_anime_bmp_map,
};

    自动化工具¶

    convert_bmp_to_rgb565.py 提供一键转换能力，后续新增或替换模式图标时可直接复用：

    cd applications/images
python3 convert_bmp_to_rgb565.py
# 输入：mode_icons_bmp.c（BMP24 格式）
# 输出：mode_icons_bmp_opt.c（RGB565 格式）

    脚本特性： - 自动解析 C 源码中的 BMP 数组和 LVGL 描述符 - 校验 BMP 魔数、位深等合法性 - 打印每张图片的转换详情和节省统计 - 生成的 C 代码可直接替换原文件

    注意事项¶

    1. LV_IMG_CF_TRUE_COLOR vs LV_IMG_CF_RAW：LVGL 对 TRUE_COLOR 格式有原生解码路径，显示性能与 RAW 一致，无额外开销
    2. 显示效果：100×100 尺寸下 RGB565 与 BMP24 肉眼无差异；若未来使用更大尺寸图片，需评估色阶损失
    3. 备份保留：原始 BMP24 文件已备份为 mode_icons_bmp.c.bak，如需回退可直接恢复
    4. 其他图片资源：desktop_bmp.c、thermal_happy_lv.c 等暂未转换，后续可参照此方案统一优化
