    AI 打印机固件 — 项目交接资料¶

    文档用途：研发 / 测试接手时的单一入口说明 — 汇总近期实机结论、代码位置、串口命令与待办验证项。
    工程路径：bk7252n_sdk-SDK_3.0.76.1_witch_lcd
    平台：BK7252N + RT-Thread + LVGL
    整理日期：2026-05-07

    关联索引：专题拆分纪要见同目录下各篇记录；总索引见 [[项目/BK7252N-AI打印机/开发记录/指南/项目开发记录总纲]]。同期口述整理亦见上一级目录 [[项目/BK7252N-AI打印机/总结/阶段性开发总结]]（与本文件要点一致，可作备份入口）。

    1. 交接摘要（接手方必读）¶

    维度 状态与结论
    打印参数 实机累计不少于约 500 张 试验；台账见 [[项目/BK7252N-AI打印机/开发记录/指南/打印机参数排列组合记录]]。文生图侧 非整幅高密度全图，图案偏稀疏，当前参数 可勉强覆盖现有客户场景。六模式端到端约 8 s～20 s，常见操作 多在十几秒量级（含业务链路与打印，不单指 GPIO 送数）。
    驱动与瓶颈 GPIO 模拟 SPI 移位送数 约在 秒级（如 384×384 量级），不是整机十几秒级的主因。步进节奏 + 加热能量（密度、脉宽、多脉冲） 才是速度与硬件风险的主轴。
    双驱动路径 thermal_printer.c 内保留 分段（legacy） 与 整行 + 多脉冲（DevTerm / whole） 两类策略；串口 prn_tune 可切换 heatmode whole \| seg、heatpulse 等（以 prn_tune help 为准）。
    硬件风险 新路径在多脉冲、偏高能量下 曾损坏 2 台设备；接手方 务必保守起步，依赖 prn_tune set stop / resume 等温控，避免无设备环境下盲目加压。
    多语言 21 种（语言 ID 0～20），与 g_device_lang_table[] 顺序绑定；串口 cfglang；维护指南见 [[项目/BK7252N-AI打印机/开发记录/指南/BK7252n-LVGL多语言开发指南]]。
    UI LVGL 可做动效，但须 控制 CPU / 内存 / 刷新；电池、WiFi、语音反馈等倾向 程序化绘图 减资源占用。当前桌面为第一版可用方案。
    OTA 代码 已接入（HTTP Shell + MQTT otaCheck 触发）；作者 未完成完整回归，需接手方 安排专项测试。

    3. 打印模块交接要点¶

    3.1 串口调参（MSH）¶

    • 前提：RT_USING_FINSH 开启；打印机已初始化；打印进行中会拒绝修改（busy printing）。
    • 命令：prn_tune help / prn_tune show。
    • 说明文档：[[项目/BK7252N-AI打印机/开发记录/指南/prn_tune串口调参命令]]。

    3.2 代码锚点¶

    内容 路径
    调参命令 applications/Printer/thermal_printer.c：prn_tune_cmd，MSH_CMD_EXPORT_ALIAS(..., prn_tune, ...)
    整行 / 分段 同文件：print_line_whole、print_line_segmented、heat_layout、whole_line_strobe_pulses
    算法综述 [[项目/BK7252N-AI打印机/开发记录/日志/2026-05-07_热敏打印机驱动算法分析与工作记录]]

    3.3 接手方建议¶

    1. 回现场有设备后，从 preset L1～L4 或文档保守组合 起步，再对比 whole + 低 heatpulse。
    2. 任何激进试验前确认 测温周期（如 prn_tune set check）与 停打温度 生效。
    3. 参考开源思路的目录（若仓库仍存在）：开源打印机代码/DevTerm/Code/thermal_printer/（以当前仓库实际路径为准）。

    4. 多语言模块交接要点¶

    项目 说明
    语言数量 21（ID 0～20，与 DEVICE_CFG_LANG_MAX_ID 一致）
    Shell cfglang —— applications/mqtt_backend/device_config.c：cfglang()，MSH_CMD_EXPORT
    语言表 同文件 g_device_lang_table[]；顺序必须与 ui_strings / UI 枚举 严格一致
    维护指南 [[项目/BK7252N-AI打印机/开发记录/指南/BK7252n-LVGL多语言开发指南]]、[[项目/BK7252N-AI打印机/开发记录/日志/2026-05-07-多语言字库与稳定性开发纪要]]
    字体生成脚本 tools/scripts/gen_lv_font_simsun_16_cjk.py、gen_lv_font_ui_i18n_14.py

    文案批量对齐建议 continues 借助工具辅助；人工逐语种比对成本高。

    5. UI 模块交接要点¶

    • 原则：特效可做，避免高开销动画与大图资源堆砌。
    • 状态图标：电池、WiFi、语音反馈等 —— LVGL 绘制为主。
    • 详细纪要：[[项目/BK7252N-AI打印机/开发记录/日志/2026-05-06_LVGL主桌面六模式与语音卡片UI移植总结]]、[[项目/BK7252N-AI打印机/开发记录/日志/2026-05-07_LVGL模式卡片资源与动效开发总结]]、[[项目/BK7252N-AI打印机/开发记录/日志/2026-05-07_LVGL桌面二维码与WS预览UI开发总结]]。

    6. OTA 交接要点（待测）¶

    6.1 入口¶

    方式 路径 / 说明
    HTTP Shell applications/OTA_Upgrade/OTA_Upgrade.c：ota_upgrade_http [url] → ota_upgrade_start_thread → ota_upgrade_download_fw
    MQTT applications/mqtt_backend/mqtt_backend_message.c：解析有效 download_url 后 ota_upgrade_start_thread
    依赖 需 PKG_USING_WEBCLIENT 与 PKG_USING_RT_OTA（见 OTA_Upgrade.c 条件编译）
    分区 config/fal_cfg.h、fal 分区与 RT_BK_DL_PART_NAME、RT_BK_APP_NAME 一致

    6.2 建议测试清单（接手方执行并勾选）¶

    • 使用可控 HTTP 固件 URL 执行 ota_upgrade_http，验证下载、进度与重启后版本号。
    • 走 MQTT otaCheck（或等价 payload） 全链路触发 OTA。
    • 弱网 / 中途断电 等异常（按项目测试规范记录日志）。
    • 失败场景保留：HTTP 状态、content_length、分区名与偏移、Flash 错误码。

    10. 接手检查清单（通用）¶

    • 已阅读 §1 摘要与 §7 风险。
    • 本地或 CI 能 完整编译 当前分支。
    • 有一份 与分区表匹配 的烧录流程说明（见烧录教程）。
    • 打印：prn_tune show baseline 存档后再做对比试验。
    • 多语言：修改语言表后执行 ID / 枚举 / 表行 三方一致性检查。
    • OTA：按 §6.2 完成或排期 专项回归。
