# LVGL 主桌面六模式与语音卡片 UI 移植总结

**日期**：2026-05-06  
**参考快照**：`applications.-wjz 参数已经写好了-0506/`  
**目标工程**：`applications/`（BK7252n + RT-Thread + LVGL）  
**编译命令**：`scons --beken=bk7252n`

---

## 1. 背景

将快照目录中与 **主桌面六种业务模式展示**、**多语言文案**、**长按语音键后的录音 / 生成等待 / 错误反馈界面** 相关的实现，合并进当前仓库的 `applications`，并保持与 `device_config` 语言 ID（0～20）一致。

---

## 2. 主桌面：六模式图标 + 多语言标题

### 2.1 行为说明

- 主桌面中间区域由原先「整屏 `desktop_modes` 大图」改为 **135×135 卡片**：当前模式展示 **100×100 `mode_icon_*` BMP** + **`ui_strings` 模式标题**（随语言切换）。
- 六种模式的英文 key（供 MQTT / `audio_upload_set_mode`）不变：`anime`、`cartoon`、`ink_wash`、`line_art`、`simple_drawing`、`sketch`。

### 2.2 新增 / 调整文件

| 路径 | 说明 |
|------|------|
| `applications/images/mode_icons_bmp.c` / `.h` | 六模式小图标资源（可由 `tools/scripts/gen_mode_icons_bmp.py` 再生成） |
| `applications/images/SConscript` | `SrcRemove(src, 'desktop_modes_bmp.c')`，避免与大图重复占 Flash |
| `applications/pages/ui_strings.c` / `.h` | 21 语言 × 全文案表（含 `UI_STR_MODE_*`、语音/错误/页脚等） |
| `applications/pages/lv_example_meter.c` | 模式槽位、`main_mode_panel_apply`、`desktop_refresh_center` 等 |
| `applications/mqtt_backend/device_config.c` / `.h` | `device_config_get_language_id()`；保存语言后调用 `lv_desktop_refresh_language()` |
| `applications/pages/lv_example_meter.h` | `lv_desktop_refresh_language()` 声明 |

### 2.3 语言刷新

- `lv_desktop_refresh_language()`：异步到 LVGL 线程，`ui_strings_sync_from_device_config()` 后重建模式槽位文案并刷新当前屏。
- `device_config.c` 内提供 **`__attribute__((weak)) void lv_desktop_refresh_language(void)`** 空实现；`lv_example_meter.c` 提供强符号覆盖，避免无 UI 时链接失败。

### 2.4 可选：模式图标微动效

- 宏 **`LV_MODE_PANEL_ICON_ANIM`**（默认 `0`）：为 `1` 时六模式图标可有 bounce/sway/透明度动画（与 0506 一致，默认关闭省 RAM）。

## 3. 二级桌面：语音卡片 UI（录音 / 生成 / 错误）

### 3.1 布局对象

- **`s_sec_voice`**：全屏透明容器，内嵌与主桌面同风格的语音卡片（135×135）。
- **`s_sec_img`**：仅用于 **WS 热敏预览图**（二进制结果下发后）。
- 去掉二级桌面上 **录音 / 生成中 / 失败** 的 BMP chroma 大图路径，改为纯 LVGL 绘制 + `ui_strings`。

### 3.2 状态与 API

- 枚举 **`voice_ui_state_t`**（定义于 `lv_example_meter.h`）：`RECORDING`、`GENERATING`、`ERR_OFFLINE`、`ERR_BUSY`、`ERR_MIC`、`ERR_UPLOAD`、`ERR_TOO_SHORT`、`ERR_TIMEOUT`、`ERR_CLOUD`、`ERR_PRINT`、`ERR_CRC`、`ERR_PREVIEW`、`ERR_QR_PREVIEW` 等。
- **`lv_desktop_voice_set_state(voice_ui_state_t)`**：任意线程可调用，内部 `lv_async_call` 派发到 LVGL，映射 `s_result_view` 并 `second_desktop_refresh_center()`。
- **`lv_desktop_request_secondary_with_voice_state(voice_ui_state_t)`**：设置 pending 后请求进入二级桌面，用于 MQTT 离线等「先进页面再刷错误卡」的场景。
- **`lv_desktop_request_show_generating_image()`** → `VOICE_UI_GENERATING`（旋转 Arc + 文案）。
- **`lv_desktop_request_show_failure_image()`** → `VOICE_UI_ERR_CLOUD`（通用云端失败文案）。

### 3.3 动效（录音验证）

- **录音态**：红点卡片；宏 **`LV_VOICE_RECORDING_USE_PULSE`**（默认 **`0`**）控制是否启用透明度脉冲。默认关闭以降低录音阶段 LVGL 负载，便于验证 ASR 是否受 UI 动画影响。
- **生成等待**：Arc 旋转动画（未在本次默认关闭）。

### 3.4 进入二级桌面逻辑（节选）

- 首次进入：若无 pending 语音状态，默认 **`VOICE_UI_RECORDING` + `UI_RESULT_BG`**，刷新语音卡片。
- 已在二级桌面时再次触发进入：重置为录音卡片（与长按清 WS 预览等行为对齐）。

## 4. 按键与业务联动

### 4.1 `button_control.c`

- **P37 长按**：若 MQTT 未连接 → `lv_desktop_request_secondary_with_voice_state(VOICE_UI_ERR_OFFLINE)`；若 `audio_upload_start()` 失败 → `VOICE_UI_ERR_BUSY`；否则 **`audio_upload_start()` → `lv_desktop_request_enter_secondary_page()`**（顺序与改 UI 前成功路径一致）。
- **P36 长按**：仍为 **SN 二维码热敏打印**；在 `audio_result_wait` 锁定或打印中时不响应（与 0506 分支一致）。

### 4.2 `audio_upload.c`

在 worker 中增加 UI 反馈（不改变 Opus / 分包格式）：

- Opus 未就绪或 reset 失败 → `VOICE_UI_ERR_MIC`
- 分包发送失败 → `VOICE_UI_ERR_UPLOAD`
- 尾包发送失败 → `VOICE_UI_ERR_UPLOAD`
- 从未发出任何分包（录音过短）→ `VOICE_UI_ERR_TOO_SHORT`

### 4.3 `audio_result_wait.c` / `.h`

- 超时 → `VOICE_UI_ERR_TIMEOUT`
- `on_result_error` → `VOICE_UI_ERR_CLOUD`
- 新增 **`audio_result_wait_on_result_abort(task_id, voice_ui_state_t reason)`**，用于打印线程创建失败等需带具体原因的场景。

### 4.4 `mqtt_backend_binary_rx.c`

- 整包 CRC 校验失败：`audio_result_wait_clear()` + `lv_desktop_voice_set_state(VOICE_UI_ERR_CRC)`
- 打印线程创建失败：`audio_result_wait_on_result_abort(..., VOICE_UI_ERR_PRINT)`

## 5. 语音识别「跑偏」：代码对比结论（备忘）

- **`applications/audio/audio_upload.c`** 与快照 **`applications.-wjz …/audio_upload.c`** 在录音循环、`opus_encode`、包头、`publish_chunk`、`audio_upload_start` 等层面 **一致**；本次主要为 **错误分支增加 `lv_desktop_voice_set_state`**。
- 若识别变差与本次改动相关，更可疑的是 **录音期间 LVGL 负载**（二级桌面重建 + 动画），而非上传字节流被改坏。
- 验证手段之一：**默认关闭录音红点脉冲**（`LV_VOICE_RECORDING_USE_PULSE=0`），对比识别效果后再决定是否恢复或继续关闭生成阶段 Arc。
