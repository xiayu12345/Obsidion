# 多语言字库与稳定性开发纪要

**工程**：`bk7252n_sdk-SDK_3.0.76.1_witch_lcd`（BK7252N + RT-Thread + LVGL）  
**摘要**：本轮围绕 UI 多语言显示缺陷修复（韩文等）、串口极限操作下的 **Data Abort** 根因分析，以及 Bug 跟踪文档落地。

---

## 1. 多语言与字库

### 1.1 现象（记录时）

| 语言 ID | 代码 | 现象 |
|--------|------|------|
| 3 | ko | 界面几乎无法正常显示文字（韩文音节大量缺失） |
| 20 | uk | 曾有「整页无字」讨论；源码侧核对 `lv_font_ui_i18n_14` 码点与乌克兰语 `ui_strings` 行可一致，需结合实机固件版本区分「缺字」与「其它运行时问题」 |
| 11 / 16 / 18 | pt / tr / cs | 偶有一两字缺失的讨论；静态核对扩展拉丁字符多在 `lv_font_ui_i18n_14` 中 |

### 1.2 根因（韩文）

- `ui_strings_font()` 对 lang **0、1、3、14**（简中、繁中、韩、日）使用 **`lv_font_simsun_16_cjk`**。  
- 旧版该字体由 `lv_font_conv` 生成时，**韩文音节（Hangul Syllables）未完整纳入子集**，例如字符串中的「?」等码点在字库中不存在，导致整句渲染失败。

### 1.3 采取的措施

- **新增脚本**：`tools/scripts/gen_lv_font_simsun_16_cjk.py`  
  - 从 `applications/pages/ui_strings.c` 抽取 **lang 0、1、3、14** 全部字面字符；  
  - 使用 **fonttools `pyftsubset`** 对 **`NotoSansCJK-Regular.ttc`**（默认 face index **2**，Sans CJK SC，含汉字+韩文+假名）做子集；  
  - 调用 **`lv_font_conv`** 生成 **`beken378/app/lvgl/src/font/lv_font_simsun_16_cjk.c`**（16px、4bpp，与现有配置一致）。  
- **环境依赖**：Node.js、`LV_FONT_CONV_JS`、`pyftsubset`（fonttools）、系统路径下的 Noto CJK TTC（可通过环境变量覆盖路径与 face index）。  
- **构建**：本地已通过 `scons --beken=bk7252n -j8` 验证编译。

### 1.4 其它脚本说明（已有）

- `tools/scripts/gen_lv_font_ui_i18n_14.py`：`ui_strings.c` 首条语言行在文件中为第 20 行，对应 **`splitlines()` 下标 `_ROW_BASE = 19`**；用于再生 **`lv_font_ui_i18n_14`**（DejaVu + Loma 等）。  
- 多语言开发说明可参考：[[项目/BK7252N-AI打印机/开发记录/指南/BK7252n-LVGL多语言开发指南]]。

### 1.5 后续建议（字库）

- 修改上述四种语言的文案后，需重新运行 **`gen_lv_font_simsun_16_cjk.py`** 再编译。  
- uk / pt / tr / cs 若实机仍异常，应用 **addr2line / 日志** 锁定具体缺码或区分 NFC/NFD，再扩展 `gen_lv_font_ui_i18n_14` 子集。

## 2. 稳定性：Data Abort 与按键场景

### 2.1 现象

- 极限按键（secondary desktop、录音起停、MQTT 收图、QR 预览、`audio_wait` 超时等）后，偶发 **`data abort`**，PC 曾落在 **`memcpy`**（须与同版本 `rtthread.map`/`addr2line` 核对）。  
- 随后 **`shutdown` 与重启**；未必表现为堆用量单调上升至 100%。

### 2.2 架构层结论（不写实现细节）

- **首要怀疑**：**单块 MQTT 二进制共享缓冲**（约 20KB）与 **零拷贝** 投递 —— 打印线程 `mqtt_prn`、LVGL 异步预览（`lv_desktop_request_show_ws_thermal_image`）、**下一趟** `result.start` 收包 **共用同一物理缓冲区**，在背靠背任务或超时与迟到数据交错时，易产生 **覆盖仍被读取的内存** → 非法访问 / `memcpy` fault。  
- **按键**：作为多条异步链路的触发器，缩短会话间隔，**放大竞态概率**。  
- **次要**：堆碎片、分配失败后未校验指针等，需与主线竞态区分排查。
