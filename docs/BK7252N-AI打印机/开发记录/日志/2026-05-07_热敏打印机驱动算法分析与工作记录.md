## 3. 算法流程简述

### 3.1 打印一行（有墨）

`print_line` 将一行按 **4 组（group）** 扫描：

- 某组有黑点：构造仅该组非零的缓存 → `shift_line` → `strobe` 加热（脉宽经 gshape / clamp）→ 组间延时。
- 每组末尾调用 `motor_step_progressive`，按 `motor_steps_per_line` 在 4 组间分配步进。
- 行尾根据**整行黑点数**调整 `line_gap_us`（稀疏行可能缩短冷却，密集行加长）。

### 3.2 整页位图

`thermal_printer_print_bitmap` 主循环：

- 按 `status_check_period_lines` 周期读 ADC、高温等待、`select_heat_us` 更新 `heat_us_cached`。
- **无墨连续行**：合并为一段，走 `fast_feed_lines_no_release` 快进。
- **有墨行**：按整行 `black_dots` 阶梯增加 `dense_boost`，再 `clamp_heat_us` 后调用 `print_line`。
- 可选按 `yield_period_lines` 插入 `rt_thread_mdelay(1)`。

### 3.3 其它要点

- 微秒延时：`delay_us` 为 NOP 循环，`us * 180` 与 CPU 主频强相关，未做硬件校准。
- 温度：`temp_from_adc` 线性近似；`paper_present()` 当前实现恒为“有纸”。

## 4. 算法层面主要问题

### 4.1 时间基准未锚定物理时间

全部 STROBE 脉宽、移位时钟半周期、电机步间隔、组间/行间间隙均依赖同一套 `delay_us`。若实际 CPU 频率或编译优化导致比例漂移，会出现**系统性偏浅/偏黑、条纹、高档位最先失效**，与仅靠 `prn_tune` 反复试参难以收敛的现象一致。

### 4.2 能量补偿多重叠加（易过补偿或冲突）

- **整行** `dense_boost`（按 `black_dots` 阶梯）。
- **每组** `prn_apply_group_gshape`（按 `group_black_dots` 减/加脉宽）。

二者都在按“密度”调节加热，缺乏统一**能量预算**约束，高密度可能出现整行加长 + 高密度段再加长 → 局部过黑、渗墨；低密度段又与缩短 `line_gap` 耦合，行为难预测。

### 4.3 行间冷却仅依赖当前行点数

稀疏行会缩短 `line_gap`，但**紧跟大块实心区之后的稀疏行**往往更需要冷却头部。当前无**前一行或滑动窗口能量**估计，易在图案边界出现深浅突变或条纹。

### 4.4 空白快进与有墨行时序不一致

`fast_feed_lines_no_release` 将行间延时限制为**最多约 4?s**（与完整 `line_gap_us` 相比极短）。同一页内空白区与图像区的**“一行”时间结构**不一致，可能造成行距、有效 DPI、走纸打滑与噪声表现与预期不符。

### 4.5 无墨段不调 `shift_line`

无黑点组跳过移位，仅步进电机。是否安全取决于头片/latch 时序；存在**段边界鬼影、串扰**类风险的验证缺口。

### 4.6 温度与保护链路薄弱

ADC→温度映射粗糙；加热基准更新间隔为按行周期采样，快扫大图时可能滞后；缺纸检测未参与策略。

### 4.7 高档位对误差敏感

L4/L5 将 `inter_group_us`、`line_cooling_us` 等压得很低，且 `clk_half_us`/`latch_us` 可为 0，对延时误差最敏感。

## 5. 改进建议（优先级）

1. **时间标定或硬件定时**：定时器/PWM/示波器标定 `delay_us`，或 STROBE/STEP/SHIFT 用硬件定时产生。
2. **单一能量模型**：以段有效点数、纸移速度、头温或滑动窗口沉积能量为输入，统一输出脉宽并设硬限幅；合并或分层 `dense_boost` 与 `gshape`，避免 double counting。
3. **行间冷却**：引入历史能量（如最近 K 行点数和）或提高头温采样率/闭环。
4. **统一空白与有墨的“一行”语义**：空白快进若保留，需单独标定等效 `line_gap` 或最小冷却，避免两套 DPI。
5. **A/B 验证**：无墨段是否必须 `shift_line(全零)`。
6. **速度档**：在标定曲线上比例缩放，而非多套孤立魔法数。
