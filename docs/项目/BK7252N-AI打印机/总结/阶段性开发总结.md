## 2. 打印机驱动：GPIO 移位 vs 步进 / 加热能量

### 2.1 结论（实机与推算一致）

- **GPIO 模拟 SPI 移位送数据** 并非整机「十几秒级」瓶颈；**384×384 量级数据传送约在秒级** 合理。  
- **打印快慢主要由步进电机节奏与加热能量（密度、脉宽、多脉冲策略）决定**，与算法文档中的 gap / heat、`dense_boost`、`gshape` 等调节维度一致。

### 2.2 开源驱动移植与 `prn_tune` 双路径

- 参考网上开源实现思路：**按行送数据 + 多脉冲打印**；仓库内保留 DevTerm 参考代码目录：`开源打印机代码/DevTerm/Code/thermal_printer/`。  
- 新驱动算法已移植；**串口 `prn_tune`** 中同时纳入 **原打印机驱动逻辑** 与 **新驱动相关可调项**（含 `preset devterm`、`heatpulse`、`heatmode whole|seg` 等，具体以 `prn_tune help` 为准）。

### 2.3 风险与停机原因（务必转告硬件同事）

- 新驱动在多脉冲、较高能量配置下 **能量偏大**，曾导致 **2 台设备损坏**；当前在南京 **无剩余设备**，故 **暂停继续加压试验**。  
- **温度检测与过热保护非常关键**；后续须在 **温度闭环可靠、能量上限保守** 的前提下再尝试激进参数，避免再次烧头片或电源应力过大。

## 3. 多国语言（21 种）

### 3.1 完成情况

- 共 **21 种语言**（语言 ID **0～20**，与 `DEVICE_CFG_LANG_MAX_ID` 一致）。  
- 维护方式：沿用 **LVGL 既有 i18n / 字体切换机制**，按 `ui_strings` 等文案 **抽取码点子集生成字库**，整体 **体积增量小**。  
- 大规模语种对齐 **适合借助 AI 批量处理**；纯人工逐条比对效率过低。

### 3.2 串口命令与代码

## 4. UI（LVGL）

- LVGL 组件丰富，可做出较多视觉效果；**可以做特效与动画，但须避免高 CPU / 内存 / 刷新开销** 的实现方式。  
- 当前桌面与二级页为 **第一版可用方案**，后续可再迭代多套视觉主题。  
- **电池、WiFi 信号、AI 语音反馈** 等图标倾向使用 **LVGL 绘制（程序化图标）**，减轻静态大图资源负担。  
- 电池与 WiFi 等与系统状态相关的接口逻辑，开发中 **借助 AI 调用现有接口快速接线** 可行。

**详细 UI 记录**：`开发记录/2026-05-06_LVGL主桌面六模式与语音卡片UI移植总结.md`、`开发记录/2026-05-07_LVGL模式卡片资源与动效开发总结.md`、`开发记录/2026-05-07_LVGL桌面二维码与WS预览UI开发总结.md`。

### 5.2 代码入口（测试清单）

| 方式 | 路径 / 说明 |
|------|-------------|
| HTTP Shell | `applications/OTA_Upgrade/OTA_Upgrade.c`：命令 **`ota_upgrade_http [url]`**，内部 `ota_upgrade_start_thread` → `ota_upgrade_download_fw` |
| MQTT 触发 | `applications/mqtt_backend/mqtt_backend_message.c`：解析到有效 `download_url` 时调用 `ota_upgrade_start_thread` |

### 5.3 建议测试步骤

1. 确认配置与分区与产物匹配后，使用 **可直连的 HTTP 固件 URL** 执行 `ota_upgrade_http`，观察下载进度与重启后版本。  
2. 再走 **MQTT `otaCheck`（或等价字段）** 下发 URL 的整条链路。  
3. 补充 **弱网、断电中途** 等异常场景（按项目测试规范）。  
4. 失败时保留日志：**HTTP 状态、`content_length`、分区名与偏移、flash 错误码**。
