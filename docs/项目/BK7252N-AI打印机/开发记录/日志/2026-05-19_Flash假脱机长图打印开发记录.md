# 2026-05-19 Flash 假脱机长图打印开发记录

## 需求背景

BK7252N 内部 RAM 仅 256KB，扣除 Wi-Fi / MQTT / LVGL / 音频等模块后，可用于打印图片的堆内存不足 30KB。MQTT `imagePrint` 接收图片使用 28KB 共享缓冲区（`MQTT_BINARY_SHARED_BUF_SIZE = 28KB`），对应 384 宽热敏位图约 597 行（~7cm 纸长），超过此长度的图片直接拒绝，无法打印。

硬件条件：
- **无 PSRAM**（`BEKEN_USING_QSPI_PSRAM_HEAP` 未启用，且硬件未焊接 PSRAM）
- Flash 4MB，OTA 分区 1632KB，日常大量闲置
- `convert_base64_to_print_data` 函数未使用（实际走 MQTT 二进制协议）
- 流式逐行打印不可行（MQTT 分包可能乱序/丢包/不完整）

## 方案选型

| 方案 | 可行性 | 原因 |
|------|--------|------|
| 启用 PSRAM | ? | 硬件无 PSRAM |
| 流式逐行打印 | ? | MQTT 分包乱序/丢包风险，无法保证完整性 |
| 增大 RAM 缓冲 | ? | 内部 RAM 已极度紧张 |
| 压缩传输 | ?? | 需服务端配合，且解压也耗 RAM |
| **Flash 假脱机** | ? | 复用 OTA 分区，零额外 RAM，与 OTA 互斥 |

## 最终方案：OTA 分区假脱机 + 双模式自适应

### 核心思路

```
MQTT 图片接收 → 判断大小
  ├── ≤28KB → 内存缓冲（快路径，和之前一样）
  └── >28KB → Flash 假脱机（OTA 分区）
                ├── 接收时逐包写 Flash
                └── 收齐后逐行从 Flash 读 48 字节打印
```

- **内存占用**：始终 48 字节行缓冲，不受图片长度限制
- **最大图片**：OTA 分区 1632KB → 384×34000 行 ≈ 4.2 米纸长
- **互斥机制**：信号量保护，打印与 OTA 互斥

### 互斥策略

```
  打印进行中 → 来了 OTA → OTA 等待最多 30 秒
  OTA 进行中 → 来了大图 → 直接拒绝（小图不受影响，不占 OTA 分区）
```

## 改动文件清单

### 1. `thermal_printer.h` — 宏开关 + 新增接口

```c
#define CFG_PRN_FLASH_SPOOL  // 功能总开关（注释掉即完全禁用）

// OTA 互斥锁
bool thermal_printer_spool_lock_try(void);      // 非阻塞尝试获取
void thermal_printer_spool_lock_release(void);   // 释放锁
bool thermal_printer_spool_is_locked(void);      // 查询锁状态

// Flash 逐行打印
int thermal_printer_print_bitmap_from_flash(uint32_t flash_offset,
                                            uint16_t width, uint16_t height);
```

### 2. `thermal_printer.c` — 锁实现 + Flash 打印函数

- 在 `thermal_printer_init()` 中初始化信号量 `s_prn_spool_sem`
- 新增 `thermal_printer_print_bitmap_from_flash()`：
  - 逻辑与 `thermal_printer_print_bitmap()` 完全一致
  - 区别：行数据通过 `bk_flash_read(BK_PARTITION_OTA, offset, line, 48)` 逐行读取
  - 包含完整的白行跳过(white-run)、温度补偿、黑点密度调节等逻辑

### 3. `mqtt_backend_binary_rx.c` — 双模式接收 + Flash 打印线程

**接收会话结构扩展**（`mqtt_backend_image_print_rx_ctx_t`）：
```c
#ifdef CFG_PRN_FLASH_SPOOL
    rt_bool_t use_flash;          // 当前会话是否走 Flash
    uint32_t flash_erase_end;     // Flash 已擦除到的末尾地址
#endif
```

**打印任务结构扩展**（`mqtt_backend_print_job_t`）：
```c
#ifdef CFG_PRN_FLASH_SPOOL
    rt_bool_t from_flash;         // 数据在 Flash 而非 RAM
    uint32_t flash_offset;        // Flash 分区内偏移
#endif
```

**关键改动点**：

1. `mqtt_backend_binary_rx_start_image_print()` — 智能路径选择：
   - `total_size > 28KB` 且 `CFG_PRN_FLASH_SPOOL` 启用 → Flash 模式
   - 获取 OTA 互斥锁 → 擦除 OTA 分区 → 设置 `use_flash = true`
   - `total_size ≤ 28KB` → 走原 RAM 路径

2. `mqtt_backend_binary_rx_feed_image_print()` — Flash 写路径：
   - Flash 模式：`bk_flash_write(BK_PARTITION_OTA, recv_size, data, len)`
   - RAM 模式：`memcpy()` 到共享缓冲区

3. `mqtt_backend_spawn_print_flash()` — Flash 打印线程创建：
   - 设置 `from_flash = true, flash_offset = 0`
   - 打印线程结束后释放 OTA 锁

4. `mqtt_backend_print_worker()` — 双模式打印：
   - Flash 模式：调用 `thermal_printer_print_bitmap_from_flash()`
   - RAM 模式：调用 `thermal_printer_print_bitmap()`
   - 退出时 Flash 模式自动释放 OTA 锁

### 4. `OTA_Upgrade.c` — OTA 互斥

```c
// ota_upgrade_thread_entry() 开头
while (thermal_printer_spool_is_locked() && wait_cnt < 600) {
    rt_thread_mdelay(50);  // 等待打印完成，最多 30 秒
}
thermal_printer_spool_lock_try();  // 获取锁，阻止后续打印

// __exit 标签处
thermal_printer_spool_lock_release();  // 释放锁
```

## 数据流对比

### 小图（≤28KB，路径不变）

```
MQTT → 28KB 共享 RAM 缓冲 → 打印线程 → thermal_printer_print_bitmap()
```

### 大图（>28KB，Flash 路径）

```
MQTT ──逐包──→ bk_flash_write(BK_PARTITION_OTA, ...)
                       │
              (全部收齐，CRC 隐含在协议层)
                       │
                       ▼
              打印线程启动（持 OTA 锁）
                       │
            for (y = 0; y < height; y++)
              bk_flash_read(OTA, y*48, line, 48);
              print_line(line, heat_us);
                       │
                       ▼
              释放 OTA 锁
```

## 注意事项

1. **仅对 `imagePrint` 命令生效**，`audioChunk`（语音生成图）仍走纯 RAM 路径
2. Flash 擦除为扇区对齐（4KB），会擦除比实际需求略多的空间
3. OTA 分区原始数据在打印后会被覆盖，下次 OTA 前服务端必须重新下发完整固件
4. 打印期间来了 OTA 指令 → OTA 等待。若打印卡死（硬件故障）最长等待 30 秒后强行走 OTA
5. 若 Flash 为 2MB（`FLASH_SELECTION_TYPE_2M`），OTA 分区仅 560KB，仍可打印约 12000 行

## 测试要点
