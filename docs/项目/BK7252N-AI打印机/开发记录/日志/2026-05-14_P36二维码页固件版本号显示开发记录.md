# P36 长按二维码页：固件版本号显示开发记录

**日期**：2026-05-14  
**工程**：`bk7252n_sdk-SDK_3.0.76.1_witch_lcd`  
**编译命令**：`scons --beken=bk7252n -j4`

---

## 1. 背景

P36 按键长按后进入静态二维码页面，此前仅显示 **SN（设备序列号）** + **二维码图片** + **底部下载提示**。  
为便于现场排查固件版本，需要在 SN 下方显示当前应用版本号，即 `MQTT_OTA_CURRENT_VERSION` 宏的值（`V0.0.1`）。

---

## 2. 改动文件清单

| 文件 | 改动类型 | 说明 |
|------|----------|------|
| `applications/mqtt_backend/mqtt_backend_client.h` | 新增宏 | 暴露 `MQTT_OTA_CURRENT_VERSION` 供 UI 层引用 |
| `applications/mqtt_backend/mqtt_backend_client.c` | 删除宏 | 移除本地定义，统一从 header 获取 |
| `applications/pages/lv_example_meter.c` | 修改 | 新增版本标签 `s_sec_qr_ver_lbl` 及配套逻辑 |

---

## 3. 详细改动

### 3.1 `mqtt_backend_client.h` — 暴露版本宏

```c
/** 当前应用固件版本号，用于 OTA 检查及界面展示 */
#define MQTT_OTA_CURRENT_VERSION "V0.0.1"
```

置于 `#ifndef __MQTT_BACKEND_CLIENT_H__` 之后、`typedef struct` 之前，供 `mqtt_backend_client.c` 和 `lv_example_meter.c` 共用。

### 3.2 `mqtt_backend_client.c` — 移除重复宏

删除原有的本地 `#define MQTT_OTA_CURRENT_VERSION "V0.0.1"`，改为从 header 继承，避免版本号分散在两处导致不一致。

### 3.3 `lv_example_meter.c` — 版本标签实现

#### 3.3.1 头文件包含

`mqtt_backend/mqtt_backend_client.h` 改为无条件包含（原先仅在 `LV_DESKTOP_STATUS_WIFI_USE_MQTT=1` 时包含），确保 `MQTT_OTA_CURRENT_VERSION` 宏始终可用。

#### 3.3.2 新增静态变量

```c
static lv_obj_t *s_sec_qr_ver_lbl;
```

与 `s_sec_qr_sn_lbl` 同级声明。

#### 3.3.3 标签创建（`ui_build_base_layers`）

在 `s_sec_qr_sn_lbl` 创建之后、`s_sec_qr_img` 之前：

```c
s_sec_qr_ver_lbl = lv_label_create(s_sec_qr_panel);
lv_label_set_long_mode(s_sec_qr_ver_lbl, LV_LABEL_LONG_DOT);
lv_obj_set_style_pad_all(s_sec_qr_ver_lbl, 0, 0);
lv_obj_add_flag(s_sec_qr_ver_lbl, LV_OBJ_FLAG_HIDDEN);
```

标签作为 `s_sec_qr_panel` 的子对象，初始隐藏。

#### 3.3.4 显示逻辑（`second_desktop_refresh_center` — `UI_RESULT_QR_STATIC` 分支）

在 SN 标签显示后、二维码图片显示前：

```c
/* 版本号：紧贴 SN 下方，不改变原有面板尺寸与二维码位置 */
lv_label_set_text(s_sec_qr_ver_lbl, "Ver: " MQTT_OTA_CURRENT_VERSION);
lv_obj_set_style_text_font(s_sec_qr_ver_lbl, LV_FONT_DEFAULT, 0);
lv_obj_set_style_text_color(s_sec_qr_ver_lbl, lv_color_hex(0x606060), 0);
lv_obj_set_style_text_align(s_sec_qr_ver_lbl, LV_TEXT_ALIGN_CENTER, 0);
lv_label_set_long_mode(s_sec_qr_ver_lbl, LV_LABEL_LONG_DOT);
lv_obj_set_width(s_sec_qr_ver_lbl, THERMAL_QR_FRAME_W - 8);
lv_obj_set_height(s_sec_qr_ver_lbl, LV_SIZE_CONTENT);
lv_obj_align(s_sec_qr_ver_lbl, LV_ALIGN_TOP_MID, 0,
             THERMAL_QR_SN_Y_OFS + THERMAL_QR_SN_ROW_H - 4);
lv_obj_clear_flag(s_sec_qr_ver_lbl, LV_OBJ_FLAG_HIDDEN);
```

关键参数：
- **字体**：`LV_FONT_DEFAULT`（与 SN 行一致）
- **颜色**：`#606060`（灰色，比 SN 的 `#101010` 浅，形成层级区分）
- **高度**：`LV_SIZE_CONTENT`（自适应内容，不额外占用固定像素）
- **Y 偏移**：`THERMAL_QR_SN_Y_OFS + THERMAL_QR_SN_ROW_H - 4` = `3 + 24 - 4 = 23`，即 SN 行底部向上 4 像素，使版本号紧贴 SN 但不占用额外间距

#### 3.3.5 层级处理

二维码图片（`s_sec_qr_img`）创建在版本标签之后，LVGL 中后创建的子对象绘制在上层，会遮挡版本文字。  
在二维码显示后调用 `lv_obj_move_foreground(s_sec_qr_ver_lbl)` 将版本标签提到最顶层，确保文字可见。

#### 3.3.6 清理与空指针检查

- 清理函数中新增 `s_sec_qr_ver_lbl = NULL`
- `second_desktop_refresh_center` 空指针检查新增 `s_sec_qr_ver_lbl == NULL`

---

## 4. 未改动项

- **`thermal_happy_lv.h`**：不新增布局常量，不调整 `THERMAL_QR_FRAME_H`。版本号使用 `LV_SIZE_CONTENT` 高度，叠在 SN 与二维码之间的自然空隙，不改变面板尺寸。
- **二维码图片位置**：`THERMAL_QR_SN_Y_OFS + THERMAL_QR_SN_ROW_H + THERMAL_QR_PREVIEW_TOP_PAD` 保持不变。
- **SN 行样式与位置**：完全不变。

---

## 5. 效果

长按 P36 进入二维码页面后，显示布局（自上而下）：

```
┌──────────────────────┐
│     SN: xxxxxxxx     │  ← 深色 #101010, Y=3, H=24
│   Ver: V0.0.1        │  ← 灰色 #606060, Y=23, 自适应高度
├──────────────────────┤
│                      │
│     [ 二维码 ]       │  ← 120×120, Y=27（位置未变）
│                      │
├──────────────────────┤
│   扫码下载 APP ...   │  ← 底部提示
└──────────────────────┘
```

版本号在 SN 正下方、二维码上方，不改变面板外框尺寸和原有各元素位置。

---

## 6. 注意事项

- 版本号字符串来源为 `MQTT_OTA_CURRENT_VERSION` 宏，OTA 检查上报和 UI 展示共用同一处定义，后续发版时只需修改 `mqtt_backend_client.h` 中一处即可。
- 版本标签使用 `LV_SIZE_CONTENT` 高度，若后续版本号字符串变长（如 `V1.0.0-beta`），标签会自动扩展高度，无需调整布局常量。
- 如版本文字与二维码顶部重叠过多，可微调 Y 偏移中的 `-4` 值（增大负数上移，减小负数下移）。
