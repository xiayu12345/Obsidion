# H133 任务与线程列表

> 主应用为 Linux 用户态多线程（pthread），非 RT-Thread。以下为 `lv_projector` 中已识别的并发模块。

## 主线程

| 线程 | 入口 | 职责 |
|---|---|---|
| main | `main.c` | LVGL 主循环 `lv_task_handler()` |

## 业务线程（pthread_create）

| 模块 | 源文件 | 职责 |
|---|---|---|
| DiskManager | `Common/disk_manager/DiskManager.c` | SD 卡 uevent 监听与挂载 |
| 媒体播放 | `Media/middle_ware/lv_pro_res_media_player_int.c` | 解码播放后台 |
| 电子书 | `Media/middle_ware/lv_pro_res_ebook_player_int.c` | 电子书播放 |
| 蓝牙 | `Bluetooth/lv_pro_res_bt.c` | BT 协议栈交互 |
| 工厂测试 | `Factory_test/factory_test.c` | 产测流程 |
| AWCast 音视频 | `AWCast/src/wfd_player/playback/*.c` | Miracast 解码渲染 |
| 设置 | `Common/setting/setting_common.c` | 设置保存后台 |

## LVGL 定时器 / 任务

LVGL 内部 `lv_timer` 驱动 UI 刷新，业务层通过 `lv_async_call` 回调 UI 线程。

## 外部进程 / 服务

| 服务 | 说明 |
|---|---|
| sshd | OpenSSH，端口 22 |
| wpa_supplicant | WiFi |
| adbd | USB 调试（可选） |
| 豆包/MQTT | 网络长连接（Layer 模块内） |

## 排查线程问题

```bash
# 板端
ps | grep lv_projector
cat /proc/$(pidof lv_projector)/status
# 崩溃栈
gdb /usr/bin/lv_projector  # 若板端有 gdb
```

## 注意

- G2D / 显示操作应在 LVGL 线程或 sunxifb 约定线程调用，跨线程直接操作 lv_obj 需 `lv_async_call`
- 媒体解码线程通过 middle_ware 与 UI 解耦
