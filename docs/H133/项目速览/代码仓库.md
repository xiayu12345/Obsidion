## 仓库形态：repo 多仓

顶层**不是**单一 Git 仓库，而是 Android **repo** 管理 dozens of 子仓：

```
HDMI/
├── .repo/
│   ├── manifests/          # 清单（default.xml）
│   └── projects/           # 子仓 gitdir 映射
├── build/                  # 子仓：Tina 构建脚本
├── brandy/brandy-2.0/      # 子仓：U-Boot
├── kernel/linux-5.4/       # 子仓：内核
├── device/                 # 子仓：板级配置
├── openwrt/                # 子仓：OpenWrt
├── platform/               # 子仓：lv_projector 等
├── tools/                  # 子仓：烧录工具
├── docs/                   # 工程内 Markdown（非独立仓也可）
├── H133开发文档-wjz/       # 文档归档（随工程）
└── out/                    # 编译产物（不入库）
```

## repo / Git 工作流

```bash
cd ~/Workspace/HDMI_new/HDMI

# 查看所有子仓改动
repo status

# 同步（注意会覆盖未提交改动）
repo sync

# 在单个子仓提交（示例：主应用）
cd platform/thirdparty/gui/lvgl-8/lv_projector
git status
git diff
git add src/main.c
git commit -m "fix: restore system_init_early for SD mount"
```

### 注意

- **不要**在顶层当普通 git 仓操作
- **不要**在 `openwrt/openwrt/` 子目录单独 `make`（丢失 `LICHEE_*` 变量）
- 改内核 / U-Boot / DTS 后走 `./build.sh kernel bootloader`
