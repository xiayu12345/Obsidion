# H133 调试相关命令

## SSH 登录

```bash
# 开发机
ssh h133-ubuntu

# 板端（同网段）
ssh root@<板子IP>    # 密码 tina
```

## 板端常用排查

```bash
# 进程
ps | grep lv_projector
killall lv_projector

# 存储
ls -l /dev/mmcblk*
cat /proc/mounts | grep mmc
ls -la /mnt/SDCARD

# 显示
cat /sys/class/disp/disp/attr/sys
cat /sys/class/disp/disp/attr/debug

# 手动挂 SD（排查用）
mount -t vfat /dev/mmcblk0p1 /mnt/SDCARD

# 内核日志
dmesg | tail -100
dmesg | grep -E 'G2D|DISP|mmc|hdmi|ion'

# 网络
ifconfig
ping -c 3 192.168.1.1
```

## G2D / 显示异常

典型日志：

```
[G2D] dma_buf_get failed, fd=255
ERROR: sunxifb_g2d_blend G2D_CMD_BLD_H failed
```

排查：

1. 确认顶层编译（非 openwrt 子目录单独 make）
2. 对比内核 `g2d_driver.h` 与用户态 `g2d_driver_enh.h` 结构体大小
3. 清理 `build_dir/target/lv_projector/` 重编

详见 [[踩坑记录/index|踩坑记录]]。

## SD 卡不挂载

1. `ls /dev/mmcblk0*` 有节点？
2. `/proc/mounts` 无 mmc → 查 `main.c` 是否调用 `system_init_early()`
3. 手动 mount 成功 → 软件问题；失败 → 文件系统/驱动

## 远程日志

```bash
# 板端输出重定向
/usr/bin/lv_projector 2>&1 | tee /tmp/lv.log

# PC 拉日志
scp root@<板子IP>:/tmp/lv.log .
```
